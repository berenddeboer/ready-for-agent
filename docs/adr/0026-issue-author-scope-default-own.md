# Default Relevant Issues to Operator GitHub User authorship

By default the Issue Reconciler includes only Ready-labeled Issues whose Issue Author matches the Operator GitHub User for that Repository. Matching is case-insensitive on login; missing or ghost authors never match. Scope uses each Issue’s own author only—parent authorship does not pull in or exclude children; hierarchy and Issue-closing PR rules are unchanged. The per-Repository setting Include all Issue Authors (default off) restores all-author eligibility. Operator GitHub User is the viewer login for the same credential path used for GitHub API access (Keymaxxer or ambient `gh`), not a separate harness account. When Include all is off and identity cannot be resolved, reconciliation fails without rewriting the Issue store for that attempt. Ready-labeled fetches include Issue Author so the store and relevance share one source. Out-of-scope Issues are removed on successful reconcile; Work Items need no special author-scope path. We rejected UI-only filtering and a separate stored operator identity because the Issue store must stay a pure projection of GitHub plus Repository settings.
