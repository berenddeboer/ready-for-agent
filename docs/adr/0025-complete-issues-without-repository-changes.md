# Complete Issues Without Repository Changes

Ready For Agent supports agent-executable Issues whose objective produces findings or GitHub-side work but no repository changes. After Implement, Assess Changes compares the worktree and branch with the Work Item's recorded starting commit; observable changes continue to Pre-Commit without consulting OpenCode, while an apparently unchanged result is confirmed by the Implement Session using the build model. A confirmed No-Change Outcome includes a concise summary and skips Pre-Commit and Review because repository quality gates have nothing to validate.

The no-change branch persists the summary, posts it once to the GitHub Issue using a Work Item marker, closes the Issue with the completed reason through GitHubService, runs local cleanup, and completes the Work Item without a pull request. Close Issue is retryable and idempotent, and an already-closed Issue is accepted after ensuring the summary exists. Auto-merge does not gate this branch because there is no pull request; Implement Locally provides operator approval by pausing before Close Issue, just as changed work pauses before Commit. Rejected alternatives were requiring an operator-selected task type, treating an empty Commit or PR as completion, running repository quality gates on unchanged work, and relying only on Git without asking the Implement Session whether the absence of changes was intentional.
