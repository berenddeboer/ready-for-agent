# Actionable Local Review Fix Loop

Local Review was exit-success only: the harness ran `/review`, ignored findings, and advanced to Commit. The same OpenCode session still held the review report, so Commit often “helpfully” implemented findings under Commit’s short budget and timed out. Review is now actionable inside the Review Lifecycle Step: a reviewing pass (review model) reports clean vs has findings; when findings exist, a build-model interpret/apply pass may fix, defer, or clear them; fixed work re-runs Pre-Commit then reviews again. At most three Review Fix Rounds per Step Run; exhaustion is Needs Human (Retry resets the counter and restarts at reviewing). Deferred or interpret-clean advances to Commit. Commit’s prompt forbids non-commit work so findings are never applied there. Rejected alternatives were a separate Apply Review Findings Lifecycle Step (extra operator surface without clearer control), advisory-only Review (wastes the review model and still leaks work into Commit), and failing the Step Run at the round limit instead of Needs Human (weaker operator signal).
