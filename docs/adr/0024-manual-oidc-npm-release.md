# Manual conventional-commit release with npm OIDC

Releases are cut only by **manual `workflow_dispatch`** on `.github/workflows/ci-cd.yml` (**CI/CD**), not on every green `main` push. Version is computed from **conventional commits** since the last tag (first public line `0.1.0`; breaking changes bump major even on 0.x). If there is nothing to release, the workflow **fails**. The release job builds platform binaries, publishes launcher + platform packages to npm with **Trusted Publishing (OIDC)** and **provenance** (no long-lived `NPM_TOKEN`), tags `vX.Y.Z`, and creates a **GitHub Release** with notes and the same binaries. Pull requests use `.github/workflows/pr.yml` (**PR**); `push` to `main` uses CI/CD quality gates (lint, knip, test, typecheck) then the live Harness e2e step (vault key never available to PR workflows). Rejected: continuous publish from every merge, free-form version input without commit analysis, and npm token secrets in Actions.
